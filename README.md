# Лабораторные работы

## 1. Ознакомление с концепциями ООП: наследование и полиморфизм типов (виртуальные методы)

Построить иерархию классов для вывода графических фигур: **отрезок, прямоугольник, эллипс** и т.д. — **не менее 6 фигур**.  
- Распределить классы по модулям.  
- Создать список фигур в виде отдельного класса.  
- В главном модуле программы добавить в список различные фигуры (статическая инициализация), после чего запустить рисование списка фигур.  
- Язык: **C++, C# или Java**.  
- Для рисования использовать любую подходящую графическую библиотеку.  

---

## 2. Графический редактор

Расширить пример с графическими фигурами так, чтобы фигуры можно было создавать на уровне пользовательского интерфейса.  
Способы создания (на выбор студента):
- ввод координат с помощью мыши,
- диалоговый ввод значений,
- ввод на скриптовом языке.

**Условия:**
- Создание объекта не должно требовать изменения существующего кода при добавлении нового класса (**никаких** `switch` / `case` / множественных `if`).
- Получившаяся программа должна быть примитивным графическим редактором.
- Классы фигур **не должны содержать метод рисования**.

---

## 3. Сериализация объектов

Выбрать предметную область и построить иерархию классов (не менее **6 классов**).  
Реализовать сериализацию/десериализацию объектов в файл / из файла.  
Формат сериализации определяется вариантом (см. таблицу).

**Функции пользовательского интерфейса:**
1. Изменение свойств объектов (редактирование).
2. Добавление / удаление объектов из списка.
3. Сериализация / десериализация списка объектов.

**Дополнительные требования:**
- Добавление новых классов не должно приводить к переписыванию существующего кода.
- Запрещено использовать `if-else`/`switch-case` и **рефлексию**.
- *Опционально:* графический интерфейс.

### Варианты форматов сериализации

| № варианта | Формат |
|------------|--------|
| 1          | XML    |
| 2          | Binary |
| 3          | Text   |
| 4          | JSON   |
| 5          | BSON   |

> **Формула выбора варианта:**
> `("порядковый номер в группе" MOD "количество вариантов") + 1`  
> (Количество вариантов = 5)

## 4. Плагины – иерархия

### Обязательная часть (8 баллов)

На основе **2-й или 3-й лабораторной работы** расширить имеющуюся иерархию новыми классами с помощью **динамической загрузки модуля (плагина)**.  
Новые модули должны добавлять или расширять функциональность базовой программы:
- новый класс в иерархии,
- функции по работе с ним,
- новые элементы в пользовательском интерфейсе для работы с новым классом.

Загрузка плагинов:
- из папки, либо
- посредством строки-параметра в главном модуле с именем модуля (возможна перекомпиляция).

> **Идеально:** добавление нового модуля должно выполняться его динамической загрузкой **без изменения кода программы**.

### Опциональная часть (10 баллов)

Сделать **подпись плагина** с последующей проверкой подписи базовой программой (время активации и целостность). Разработать механизм подписывания плагинов.

## 5. Плагины – функциональность

### Обязательная часть (8 баллов)

На базе предыдущей лабораторной работы (№4) на основе плагинов (2–3 плагина) реализовать возможность обработки структур **перед сохранением в файл** и **после загрузки из файла**.  
Тип обработки задаётся вариантом (см. таблицу).  
Дополнительная функциональность появляется в меню настроек и зависит от загруженных плагинов.  
Загрузка плагинов производится автоматически из папки либо выбором файла плагина через пользовательский интерфейс.

### Дополнительное задание (10 баллов)

Предусмотреть дополнительную настройку функциональности плагина в меню настройки плагинов (например, параметры шифрования/архивации, выбор алгоритма, правила трансформации, кодировка и т.д.).

### Варианты обработки

| № варианта | Функциональность                                |
|------------|-------------------------------------------------|
| 1          | Трансформация XML данных в JSON                 |
| 2          | Архивация                                       |
| 3          | Шифрование / дешифрование                       |
| 4          | Трансформация XML (можно XSLT)                  |
| 5          | Сохранение контрольной суммы                    |

> **Формула выбора варианта:**  
> `("порядковый номер в группе" MOD "количество вариантов") + 1`  
> (Количество вариантов = 5)

## 6. Паттерны

На базе предыдущей лабораторной работы (№5) **обменяться с товарищем функциональными плагинами** (минимум одним) и адаптировать их с помощью паттерна **Адаптер** (появятся новые функции от плагина товарища, загруженные через плагин с адаптером).  

**Также необходимо реализовать 2 любых паттерна** в программе, пояснив уместность их использования.
